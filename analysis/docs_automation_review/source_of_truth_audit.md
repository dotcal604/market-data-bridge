## Branch A: `automate-docs-generation` (Docusaurus + TypeDoc)

### What it reuses correctly
- **TypeDoc → `src/`**: Generates TypeScript API docs from actual source. ✅ Correct source.
- **Mermaid diagrams**: New hand-written `.mmd` files. ❌ Not extracted from source — parallel content.

### What it does NOT reuse
- **MCP tool docs**: No MCP tool extraction at all. TypeDoc generates TypeScript module docs, not tool-level API reference. The 152 `server.tool()` registrations are invisible. ❌ **Critical gap**.
- **OpenAPI spec**: Not referenced. No REST API docs from `openapi-chatgpt.json`. ❌ **Critical gap**.
- **Existing guide docs** (`docs/01-06*.md`): Not included in the Docusaurus site. ❌ Missed.
- **Existing Mintlify MDX**: 28 rich files ignored. ❌ Missed.
- **Architecture diagram**: Architecture page is a new 280-line hand-written Mermaid doc. Does not reference `architecture-diagram.html`. ❌ Parallel content.

### Verdict
**Branch A creates a parallel docs universe.** TypeDoc generates raw TypeScript class/function documentation that has near-zero value for users of an MCP server. The two things users need — MCP tool reference and REST API reference — are absent. Hand-written Mermaid diagrams duplicate the existing architecture diagram. The existing 60,000 lines of guide content are ignored.

## Branch B: `docs-automation-mvp` (Zero-dependency generator)

### What it reuses correctly
- **MCP tools from `src/mcp/server.ts`**: Regex-extracts all 152 `server.tool()` calls with names, descriptions, and Zod parameters. ✅ Correct source, verified at build time (152/152 match).
- **OpenAPI spec from `openapi-chatgpt.json`**: Copies verbatim, renders via Redoc CDN. ✅ Correct source, identical output.
- **Guide docs from `docs/01-06*.md`**: Converts all 6 numbered guides to HTML. ✅ Correct source.
- **Architecture diagram from `architecture-diagram.html`**: Embeds via iframe. ✅ Correct source.

### What it does NOT reuse
- **Mintlify MDX files** (`docs/mcp/*.mdx`, `docs/workflows/*.mdx`, `docs/frontend/*.mdx`): 28 files with rich hand-written content are ignored. The `stripMdx()` function exists but is only applied to the 01-06 guides. ⚠️ Gap, but defensible as MVP scope.
- **`docs.json` navigation hierarchy**: Not read. The generated site builds its own nav. ⚠️ Minor — the Mintlify config was never deployed anyway.
- **`SYSTEM_CARD.md`, `ROADMAP.md`, `README.md`**: Not included. ⚠️ Could be added later.

### Risks
1. **Regex MCP extraction is fragile**: Line-by-line scanning with 50-line lookahead. Will break if code formatting changes. Moderate risk.
2. **Custom markdown parser**: 160 lines, handles basics but missing nested lists, blockquotes, images. Moderate risk for the 60K-line guides.
3. **Hardcoded `/market-data-bridge/` base path**: Every link assumes GitHub Pages at this path. Inflexible.
4. **Hardcoded "135" action count**: Landing page says "135 agent actions" instead of reading from OpenAPI spec.
5. **CDN dependency for Redoc**: REST API page loads Redoc from CDN at runtime. If CDN is down, API docs break.

### Verdict
**Branch B correctly identifies and reuses the repo's real sources of truth.** It reads from `server.ts`, `openapi-chatgpt.json`, `docs/01-06*.md`, and `architecture-diagram.html`. It does not create parallel content. The gaps (Mintlify MDX, SYSTEM_CARD) are defensible MVP scope.
