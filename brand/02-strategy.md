# Brand Strategy — market-data-bridge

## Positioning (one sentence)

market-data-bridge is the structured data layer that connects brokerage infrastructure to AI assistants, normalizing multi-source market data into tool-ready intelligence.

## Value Proposition

Multiple messy inputs (broker APIs, CSV exports, delayed feeds, real-time streams) go in. Clean, enriched, scored, protocol-compliant data comes out — ready for Claude, ChatGPT, or any tool consumer. The bridge doesn't trade. It doesn't advise. It makes data usable.

## Brand Personality

- **Precise**: Not approximate. Schema-validated. Type-safe.
- **Structural**: Concerned with how data connects, not how it looks.
- **Quiet confidence**: Doesn't shout. Works correctly under pressure.
- **Infrastructure-grade**: Feels like a tool, not a product pitch.
- **Transparent**: Discloses source, shows routing, never hides the plumbing.

## Visual Keywords

1. **Convergence** — multiple inputs merging to unified output
2. **Schema** — structured, grid-aligned, column-precise
3. **Bridge span** — connection across a gap (abstract, not literal)
4. **Enrichment** — raw → processed, thin → thick, simple → annotated
5. **Protocol** — formal, typed, interface-defined
6. **Alignment** — things lining up, columns matching, rows mapping
7. **Throughput** — data flowing, not static

## Anti-Patterns (must avoid)

1. **Consumer fintech polish**: Rounded corners, gradient cards, friendly illustrations. This is dev infrastructure.
2. **Crypto aesthetic**: Neon, dark-only, angular geometric noise. This is equity/options trading.
3. **"AI" signaling**: Neural networks, brain imagery, sparkle effects. The AI is the consumer, not the product.
4. **Generic SaaS**: Blue gradient, sans-serif wordmark, abstract swoosh. Must be recognizable as THIS product.
5. **Complexity masking simplicity**: If the mark needs explanation, it failed. The concept should be immediate at 32x32.

## Color Direction

Primary palette should evoke:
- **Data / infrastructure**: Cool neutrals, slate, graphite
- **Signal / active state**: A single accent that reads as "active data" — not decorative, functional
- **The existing codebase uses**: emerald (#10b981) for positive, purple (#8b5cf6) for Claude, amber (#f59e0b) for Gemini, red for negative

Proposal: Build from the codebase's own emerald as primary accent (it's already the "good signal" color), with slate/graphite structure. This roots the brand in the actual product rather than imposing an external palette.

## Typography Direction

Monospace or semi-monospace for the wordmark. This is a tool for people who live in terminals and IDEs. A humanist sans-serif would feel disconnected from the audience.
